# LLM Flux Gateway - 使用场景文档

## 首次配置流程

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        首次配置 - 从零开始                              │
└─────────────────────────────────────────────────────────────────────────┘

步骤 1: 配置厂商 (Vendors)
├─ 操作: 编辑 config/vendors.yaml 或使用默认配置
├─ UI: http://localhost:3000/#/vendors
├─ 点击: "Sync from YAML" 按钮
├─ 结果: 厂商和模型列表同步到数据库
└─ 耗时: 约 10 秒

步骤 2: 创建资产 (Assets)
├─ UI: http://localhost:3000/#/assets
├─ 点击: "Add Asset" 按钮
├─ 步骤 2.1: 选择厂商 (OpenAI / Anthropic / Google / ...)
├─ 步骤 2.2: 配置 API Key 和资产名称
├─ 步骤 2.3: 选择可用模型 (支持多选)
├─ 点击: "Create Asset"
└─ 结果: 资产创建完成，可以看到预算和余额

步骤 3: 配置路由 (Routes)
├─ UI: http://localhost:3000/#/routes
├─ 点击: "Add Route" 按钮
├─ 输入: 路由名称 (如 "生产环境路由")
├─ 选择: 关联的资产 (从下拉菜单选择)
├─ 配置: 转发规则 (可选，用于模型重写)
└─ 结果: 路由配置完成

步骤 4: 生成访问密钥 (API Keys)
├─ UI: http://localhost:3000/#/keys
├─ 点击: "Generate Key" 按钮
├─ 输入: 客户端名称 (如 "生产应用 A")
├─ 关联: 选择路由 (支持多选，设置优先级)
└─ 结果: 生成 sk-flux-xxx 格式的密钥

步骤 5: 测试验证
├─ 方法 1: 在 Assets 页面点击 "Quick Test"
├─ 方法 2: 在 Playground 页面手动测试
├─ 方法 3: 使用 curl 命令测试
└─ 验证: 确认请求正常转发

总耗时: 约 2-3 分钟 (不含 API Key 配置时间)
```

## 二次使用流程 (最常见) - 为新应用生成密钥

```
┌─────────────────────────────────────────────────────────────────────────┐
│                   二次使用 - 为新应用生成密钥  ⭐ 最常见               │
└─────────────────────────────────────────────────────────────────────────┘

场景: 给新的应用/客户端提供访问权限（路由已配置好）

步骤 1: 生成新密钥
├─ UI: http://localhost:3000/#/keys
├─ 点击: "Generate Key" 按钮
├─ 输入: 客户端/应用名称 (如 "移动应用 v2.0", "后端服务 B")
└─ 结果: 生成新密钥

步骤 2: 关联现有路由
├─ 选择: 已配置好的路由
├─ 设置: 优先级 (数字越大优先级越高)
└─ 保存: 密钥立即可用

步骤 3: 复制密钥给开发者
├─ 复制: sk-flux-xxx 格式
└─ 使用: 开发者在应用中使用

总耗时: 约 20 秒

典型场景:
├─ 新的应用需要访问 LLM
├─ 新的开发环境
├─ 新的客户端
└─ 临时测试密钥
```

## 二次使用流程 (较少见) - 添加新的 API 供应商

```
┌─────────────────────────────────────────────────────────────────────────┐
│                   二次使用 - 添加新的 API 资产                        │
└─────────────────────────────────────────────────────────────────────────┘

场景: 需要接入新的 LLM 供应商（如已有 OpenAI，现需添加 Anthropic）

步骤 1: 创建新资产
├─ UI: http://localhost:3000/#/assets
├─ 点击: "Add Asset"
├─ 选择: 厂商 (如 Anthropic)
├─ 输入: API Key
└─ 选择: 可用模型

步骤 2: 创建路由
├─ UI: http://localhost:3000/#/routes
├─ 点击: "Add Route"
├─ 输入: 路由名称
└─ 选择: 刚创建的资产

步骤 3: 关联到密钥
├─ 编辑现有密钥
├─ 添加新路由
└─ 保存

总耗时: 约 1-2 分钟

何时需要:
├─ 接入新的 LLM 供应商
├─ 创建独立的测试环境
└─ 按团队/项目隔离资产
```

## 日常使用流程

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        日常操作                                        │
└─────────────────────────────────────────────────────────────────────────┘

场景 1: 查看使用情况
├─ UI: Analytics 页面
├─ 查看: Token 使用量、请求量、成本
└─ 操作: 无需配置，仅查看

场景 2: 测试新模型
├─ UI: Assets 页面 → 点击资产卡片 → "Quick Test"
├─ 跳转: 到 Playground 页面
└─ 测试: 选择模型 → 发送请求

场景 3: 调整路由规则
├─ UI: Routes 页面 → 编辑路由
├─ 修改: 转发规则 (如模型重写)
└─ 保存: 实时生效

场景 4: 余额不足处理
├─ UI: Assets 页面 → 编辑资产
├─ 操作: 增加 Balance 或充值
└─ 结果: 资产自动恢复 Active 状态
```

## 可以省略的步骤

### 首次配置中可省略的步骤

```
┌─────────────────────────────────────────────────────────────────────────┐
│                       ⚠️  可选步骤                                     │
└─────────────────────────────────────────────────────────────────────────┘

✗ 可以省略: 编辑 YAML 配置文件
├─ 原因: 系统已内置主流厂商 (OpenAI / Anthropic / Google)
├─ 默认: 包含 30+ 常用模型
└─ 操作: 直接在 Vendors 页面点击 "Sync from YAML" 即可

✗ 可以省略: 配置路由转发规则 (Overrides)
├─ 原因: 默认直接转发，无需规则配置
├─ 场景: 仅在需要模型重写、参数覆盖时才需要
└─ 操作: Routes 页面创建路由时留空即可

✗ 可以省略: 设置预算 (Budget)
├─ 原因: 仅用于使用统计和限制
├─ 场景: 不限制使用时可以不设置
└─ 操作: 创建资产时 Budget 留空或设为 0

✗ 可以省略: 关联多个路由到密钥
├─ 原因: 一个密钥关联一个路由即可使用
├─ 场景: 仅在需要 A/B 测试或灰度发布时需要
└─ 操作: Keys 页面关联主路由即可

✗ 可以省略: 手动测试
├─ 原因: 配置正确的情况下可直接使用
├─ 建议: 首次配置后测试一次即可
└─ 操作: Assets 页面点击 "Quick Test" 按钮
```

### 推荐的最简配置流程

```
┌─────────────────────────────────────────────────────────────────────────┐
│                   🚀 最简配置 (5 分钟完成)                            │
└─────────────────────────────────────────────────────────────────────────┘

1️⃣  同步厂商 (10 秒)
   └─ Vendors → "Sync from YAML"

2️⃣  创建资产 (1 分钟)
   └─ Assets → "Add Asset" → 选择厂商 → 输入 API Key → 选择模型

3️⃣  创建路由 (30 秒)
   └─ Routes → "Add Route" → 输入名称 → 选择资产 (不配置规则)

4️⃣  生成密钥 (20 秒)
   └─ Keys → "Generate Key" → 输入名称 → 关联路由

5️⃣  复制密钥测试 (10 秒)
   └─ 复制 sk-flux-xxx 密钥 → 在应用中使用

总耗时: 约 2 分钟实际操作时间
```

## 典型使用案例

### 案例 1: 个人开发者

```
需求: 统一管理多个 LLM 供应商的 API

配置:
├─ 1 个 OpenAI 资产 (GPT-4o, GPT-4 Turbo)

├─ 灵活切换模型

```

```

配置:
├─ 1 个 Google Gemini 资产 (大额度)
├─ 1 个路由 (按模型转发)
└─ N 个密钥 (每人一个)

优势:
├─ 统一配额管理
├─ 使用记录可追溯
├─ 预算控制
└─ 按需分配权限

├─ 2 个资产 (OpenAI + Anthropic)

└─ 1 个密钥 (关联 2 个路由，设置优先级)

优势:
├─ 无需修改代码

├─ 实时对比效果

```

## 常见问题

### Q: 必须配置 Vendors 吗？
A: 不必须。首次使用时点击 "Sync from YAML" 即可，后续无需再配置。

### Q: 一个资产可以关联多少个路由？
A: 无限制。一个资产可以被多个路由使用。

### Q: 一个密钥可以关联多少个路由？
A: 无限制。支持多路由优先级配置。

### Q: 如何切换模型？
A: 方法 1: 在应用请求中指定 model 参数
       方法 2: 配置路由转发规则自动重写

### Q: 余额不足会怎样？
A: 资产状态自动变为 "Exhausted"，请求会被拒绝。增加余额后自动恢复。

### Q: 如何备份数据？
A: 数据库文件: data/database.sqlite
       配置文件: config/vendors.yaml

### Q: 可以迁移到其他服务器吗？
A: 可以。复制数据库文件和配置文件即可。
